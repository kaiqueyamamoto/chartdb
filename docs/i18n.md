# Internacionalização (i18n)

## Setup

O ChartDB usa **i18next v23** com **react-i18next** para suporte a múltiplos idiomas.

**Arquivos:**
- `src/i18n/index.ts` — configuração do i18next
- `src/i18n/locales/` — arquivos de tradução por idioma

O idioma é detectado automaticamente pelo plugin `i18next-browser-languagedetector`.

## Uso nos Componentes

```typescript
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();

  return <button>{t('toolbar.save')}</button>;
}
```

### Interpolação

```typescript
// Chave: "table.field_count": "{{count}} field(s)"
t('table.field_count', { count: fields.length })
```

### Pluralização

```typescript
// i18next trata pluralização automaticamente com sufixos _one, _other
// "items_one": "{{count}} item"
// "items_other": "{{count}} items"
t('items', { count: 5 })  // → "5 items"
```

## Estrutura dos Arquivos de Tradução

Cada locale tem um arquivo `translation.json`:

```
src/i18n/locales/
├── en/translation.json   ← arquivo de referência
├── pt_BR/translation.json
├── es/translation.json
└── ...
```

O arquivo `en/translation.json` é a fonte da verdade. Novas chaves devem ser adicionadas em inglês primeiro e depois traduzidas nos demais locales.

---

## Contribuindo com Traduções

Para adicionar ou corrigir uma tradução:

1. Localize a chave em `src/i18n/locales/en/translation.json`
2. Adicione/corrija o valor no arquivo do idioma correspondente
3. Não deixe chaves faltando — i18next faz fallback para `en` automaticamente se a chave não existir no idioma atual
